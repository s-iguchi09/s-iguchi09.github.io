WPF 標準コントロールデモアプリ › ListBox

ListBox List

ListBox はスクロール可能なリストからアイテムを選択できるコントロールです。単一・複数・拡張選択モードをサポートし、データコレクションへのバインドが可能です。

概要

WPF の ListBox は Selector を継承したリストコントロールです。ItemsSource にコレクションをバインドすることで、動的なアイテムリストを表示できます。ItemTemplate でアイテムの表示方法をカスタマイズできます。

SelectionMode プロパティで選択の挙動を制御します。Single は一度に 1 つだけ選択、Multiple は Ctrl キーなしでクリックするたびに選択を切り替え、Extended は Shift+クリックで範囲選択・Ctrl+クリックで個別追加選択が可能です。

SelectedItems（複数選択モード）や SelectedItem（単一選択モード）を ViewModel にバインドすることで、選択されたアイテムを取得できます。複数選択のバインドは直接は難しいため、IsSelected を各アイテムの ViewModel プロパティにバインドするか、ビヘイビアを使うのが一般的です。

大量のアイテムを表示する場合、デフォルトで VirtualizingStackPanel がアイテムホストとして使用されるため、表示領域外のアイテムは仮想化されてメモリを節約できます。

画面キャプチャ

デモしているプロパティ

以下のプロパティが WPF 標準コントロールデモアプリでインタラクティブにデモされています。

プロパティ設定値説明
SelectionModeSingle / Multiple / Extended選択モードを設定します。Single は 1 アイテムのみ、Multiple はクリックで独立切り替え、Extended は Shift/Ctrl+クリックで範囲・個別選択が可能です。
DisplayMemberPath (ItemsControl)string各アイテムオブジェクトの表示テキストとして使うプロパティ名を設定します。ItemTemplate なしで簡単なテキストリストを表示する場合に使います。
SelectedIndex (Selector)int選択されているアイテムのゼロ始まりインデックスを取得・設定します。-1 は未選択を意味します。
SelectedItem (Selector)object選択されているアイテムオブジェクトを取得・設定します。TwoWay バインドで ViewModel と連動させます。
SelectedValuePath / SelectedValue (Selector)string / objectSelectedValuePath で指定したプロパティ値を SelectedValue として取得します。
IsSelected (ListBoxItem)bool個々のリストアイテムが選択されているかどうかを設定します。ItemContainerStyle で TwoWay バインドします。

XAML 使用例

<ListBox ItemsSource="{Binding Countries}"
         DisplayMemberPath="Name"
         SelectedItem="{Binding SelectedCountry, Mode=TwoWay}"
         SelectionMode="Extended"
         ScrollViewer.HorizontalScrollBarVisibility="Disabled">
</ListBox>

<!-- カスタム ItemTemplate -->
<ListBox ItemsSource="{Binding Files}"
         SelectionMode="Single"
         SelectedItem="{Binding SelectedFile, Mode=TwoWay}">
  <ListBox.ItemTemplate>
    <DataTemplate>
      <StackPanel Orientation="Horizontal">
        <Image Source="{Binding Icon}" Width="16" Height="16" />
        <TextBlock Text="{Binding Name}" Margin="8,0,0,0" />
      </StackPanel>
    </DataTemplate>
  </ListBox.ItemTemplate>
</ListBox>

主な使用例

  • 国・都市の選択リスト — スクロールリストから一つの選択肢を選ぶシンプルな単一選択 UI に使用します。
  • ファイル一覧 — ファイル名のリストを表示し、選択されたファイルの内容を右ペインに表示する 2 ペイン UI に使用します。
  • 複数選択フィルター — Extended モードでカテゴリやタグを複数選択してデータをフィルタリングします。
  • 選択履歴の表示 — 最近使ったファイルや検索履歴をリスト表示し、再選択できる UI を実装します。
  • 設定項目の選択 — テーマカラーや言語設定など、複数の選択肢から一つを選ぶ設定 UI に使用します。

ヒントとベストプラクティス

  • SelectedItem を TwoWay バインド — 選択アイテムを ViewModel で管理するには SelectedItem に Mode=TwoWay を指定します。
  • 大量データには仮想化を活用 — ListBox はデフォルトで VirtualizingStackPanel を使用します。カスタム ItemsPanel を設定する場合は仮想化が失われないよう注意します。
  • ComboBox との使い分け — 常に選択肢を表示しておく必要がある場合は ListBox、スペースを節約したい場合は ComboBox を使います。
  • 空リストの表示 — ItemsSource が空の場合に「データがありません」などのメッセージを表示するには、ItemsControl の HasItems プロパティをトリガーに使います。
  • スクロールバーのカスタマイズ — ScrollViewer 添付プロパティで水平・垂直スクロールバーの表示を個別に制御できます。

関連コントロール

  • ComboBox — スペースを節約したい場合はドロップダウン形式の ComboBox を使います。
  • ListView — GridView を使った表形式の表示が必要な場合。
  • DataGrid — 複数列の表形式表示と編集が必要な場合。
