Menu Menu

Menu は階層的なコマンドメニューを表示するコントロールです。アプリウィンドウの上部に水平メニューバーとして配置されるのが一般的で、サブメニューや区切り線、アクセスキー、ショートカットキーの表示をサポートします。

概要

WPF の Menu は MenuBase を継承したコントロールで、MenuItem の階層構造からなるコマンドメニューを提供します。アプリウィンドウの最上部に水平バーとして配置し、ファイル・編集・表示などのトップレベルメニューを並べるのが一般的なパターンです。

各 MenuItem は Header にメニューテキストを、Command に実行する ICommand を設定します。トップレベルの MenuItem はホバーでサブメニューを展開し、サブメニューの MenuItem はクリックでコマンドを実行します。Icon プロパティでメニューアイテムの左側にアイコンを表示できます。

IsCheckable を True に設定した MenuItem はチェックボックス付きのトグルメニューアイテムになり、IsChecked でオン/オフ状態を管理します。「表示 > ツールバーを表示」のようなトグル可能なメニューに使用します。

InputGestureText プロパティでキーボードショートカットの表示テキスト（例：Ctrl+S）を設定できます。実際のショートカット処理は KeyBinding で別途設定しますが、InputGestureText で MenuItem の右端にショートカット文字列を表示します。

画面キャプチャ

デモしているプロパティ

以下のプロパティが WPF 標準コントロールデモアプリでインタラクティブにデモされています。

プロパティ設定値説明
IsMainMenuboolTrue のとき、ウィンドウのメインメニューとして登録されます。Alt キーでメニューにフォーカスが移動します。
IsCheckable (MenuItem)boolTrue にするとメニューアイテムがチェック可能なトグルメニューになります。
IsChecked (MenuItem)boolIsCheckable=True の場合に、メニューアイテムがチェックされているかどうかを設定します。
Command (MenuItem)ICommandクリック時に実行するコマンドを設定します。CanExecute が false の場合、メニューアイテムは自動でグレーアウトします。
Icon (MenuItem)objectメニューアイテムの左端に表示するアイコンを設定します。通常は小さな Image を配置します。
InputGestureText (MenuItem)stringメニューアイテムの右端に表示するキーボードショートカットの説明テキストを設定します（例: Ctrl+S）。

XAML 使用例

<DockPanel>
  <Menu DockPanel.Dock="Top" IsMainMenu="True">
    <MenuItem Header="ファイル(_F)">
      <MenuItem Header="新規作成(_N)" Command="ApplicationCommands.New"
                InputGestureText="Ctrl+N">
        <MenuItem.Icon>
          <Image Source="/Icons/new.png" Width="16" Height="16" />
        </MenuItem.Icon>
      </MenuItem>
      <MenuItem Header="開く(_O)" Command="ApplicationCommands.Open"
                InputGestureText="Ctrl+O" />
      <MenuItem Header="保存(_S)" Command="{Binding SaveCommand}"
                InputGestureText="Ctrl+S" />
      <Separator />
      <MenuItem Header="終了(_X)" Command="{Binding ExitCommand}" />
    </MenuItem>
    <MenuItem Header="表示(_V)">
      <MenuItem Header="ツールバーを表示" IsCheckable="True"
                IsChecked="{Binding IsToolBarVisible}" />
    </MenuItem>
  </Menu>
</DockPanel>

主な使用例

  • アプリのメインメニューバー — ファイル・編集・表示などのカテゴリ別コマンドメニューをウィンドウ上部に配置します。
  • コンテキストに応じたコマンド — 選択状態や編集モードに応じて MenuItem の IsEnabled を CanExecute で自動制御します。
  • トグル可能なビュー設定 — ツールバー・ステータスバーの表示/非表示を IsCheckable な MenuItem でトグルします。
  • 最近使ったファイル — 「ファイル」メニューの下部にバインドで動的生成する最近使ったファイルリストを追加します。
  • サブメニュー階層 — 「表示 > ズーム > 拡大」のような 3 段階のネストしたサブメニューを実装します。

ヒントとベストプラクティス

  • Command バインドを使う — MenuItem.Click イベントよりも Command バインドを使い、CanExecute で自動的に有効/無効を制御します。
  • アクセスキーは _ で設定 — Header テキストで _ の次の文字がアクセスキーになります。Alt を押すと下線が表示されます。
  • ショートカットキーは KeyBinding で設定 — InputGestureText はあくまで表示用です。実際のショートカット処理は Window.InputBindings に KeyBinding を追加して実装します。
  • Separator で論理グループを分ける — 関連するコマンドグループは Separator で区切ってメニューの可読性を向上させます。
  • ToolBar との組み合わせ — よく使うコマンドはメニューとツールバーの両方に配置し、目的に応じて使い分けられるようにします。

関連コントロール

  • ToolBar — よく使うコマンドをアイコンボタン形式で提供するツールバー。
  • Button — コンテキストメニューや画面内のコマンドには Button を使います。
