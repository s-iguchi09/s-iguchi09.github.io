WPF 標準コントロールデモアプリ › Button

Button Inputs

Button は WPF で最もよく使われるコマンドトリガーコントロールです。押されると Click イベントが発火し、ICommand インターフェイスを通じたコマンドバインドをサポートします。ButtonBase を継承しており、ClickMode やコマンド関連のプロパティが利用可能です。

概要

WPF の Button コントロールは、デスクトップアプリにおけるユーザーアクションのトリガーとして最も基本的な役割を担います。 ButtonBase を継承し、さらに ContentControl を拡張しているため、Button の中にはテキストだけでなく 画像・アイコン・複雑なパネルなど任意の WPF 要素を配置できます。

MVVM アーキテクチャにおいて特に重要なのが、Command プロパティによる ICommand インターフェイスのサポートです。 コマンドをバインドすると、ICommand.CanExecute が false を返すたびに WPF が自動でボタンを無効化し、 条件が変化すると再び有効にします。これによりコードビハインドでの有効・無効切り替えロジックが不要になります。

ButtonBase から継承された ClickMode プロパティは、Click イベントがいつ発火するかを制御します。 デフォルトの Release はマウスボタンを離したとき、Press は押し下げた瞬間、 Hover はホバー時に発火します。

IsDefault と IsCancel の 2 つの bool プロパティはダイアログの実装を大幅に簡略化します。 IsDefault="True" で Enter キー、IsCancel="True" で Escape キーとボタンを関連付けられます。

デモしているプロパティ

以下のプロパティが WPF 標準コントロールデモアプリでインタラクティブにデモされています。 アプリ上でリアルタイムに各プロパティを変更してその動作を確認できます。

プロパティ設定値説明
IsCancel bool True に設定すると、Escape キー押下でこのボタンの Click が発生します。ダイアログのキャンセルボタンに使用します。Window.DialogResult = false と組み合わせると、標準的なキャンセル動作を実現できます。
IsDefault bool True に設定すると、他の要素が Enter を処理しない場合にこのボタンの Click が発生します。フォームの「OK」ボタンや「検索」ボタンに使用し、Enter キーで送信できるようにします。
ClickMode (ButtonBase) Release / Press / Hover Click イベントが発生するタイミングを制御します。Release はマウスボタンを離したとき（デフォルト）、Press は押し下げた瞬間、Hover はカーソルが乗ったときに発火します。
IsPressed (ButtonBase) bool (ReadOnly) ボタンが現在押されているかどうかを示す読み取り専用プロパティです。ControlTemplate 内でこの値をトリガーにして、押し込んだ際の視覚的変化（色変更・スケール縮小など）を実装できます。
Command (ButtonBase) ICommand ボタンクリック時に実行するコマンドを設定します。ViewModel の ICommand プロパティをバインドします。CanExecute が false の場合、ボタンは自動で無効化されます。
CommandParameter (ButtonBase) object コマンド実行時に渡すパラメータを設定します。同一のコマンドを複数のボタンで共有し、パラメータで動作を切り替える際に活用します。

XAML 使用例

MVVM バインドと IsDefault/IsCancel を組み合わせた典型的なダイアログボタンの例です：

<StackPanel Orientation="Horizontal" HorizontalAlignment="Right">
  <Button Content="OK" Width="80" Margin="0,0,8,0"
          IsDefault="True"
          Command="{Binding OkCommand}" />
  <Button Content="キャンセル" Width="80"
          IsCancel="True"
          Command="{Binding CancelCommand}" />
</StackPanel>

<!-- ClickMode の例 -->
<Button Content="ホバーで発火" ClickMode="Hover"
        Command="{Binding HoverCommand}" />

主な使用例

  • ダイアログの OK/キャンセル — IsDefault と IsCancel を使ってキーボード操作に対応したダイアログボタンを実装します。
  • フォームの送信 — IsDefault="True" を設定することで Enter キーによるフォーム送信を実現します。
  • MVVM コマンドバインド — Command プロパティで ViewModel の ICommand をバインドし、CanExecute による自動有効/無効を活用します。
  • リッチコンテンツボタン — ContentControl の特性を活かし、アイコン＋テキストや複数行のコンテンツを持つボタンを作成します。
  • ツールバーアクション — ツールバー内に配置し、アイコンのみのシンプルなアクションボタンとして使用します。

ヒントとベストプラクティス

  • コードビハインドの Click イベントより Command を優先 — MVVM パターンでは Click イベントをコードビハインドで処理するよりも、ICommand バインドを使うとテストしやすく疎結合な設計になります。
  • CanExecute を活用する — 入力検証や処理中フラグを CanExecute に反映させると、ボタンの有効/無効が自動で更新され、余分なロジックが不要になります。
  • コンテンツには文字列以外も可能 — テキストだけでなく、Image や StackPanel を Content として設定すると、アイコン付きボタンやリッチレイアウトのボタンを簡単に作れます。
  • ToolTip を設定する — アイコンのみのボタンには必ず ToolTip を設定し、ユーザーがホバーで機能を確認できるようにしましょう。
  • アクセシビリティ対応 — コンテンツが画像の場合は AutomationProperties.Name を設定してスクリーンリーダーが読み上げられるようにします。

関連コントロール

  • ToggleButton — クリックするたびに押下状態を保持するトグルボタン。CheckBox と RadioButton の基底クラス。
  • RepeatButton — 押し続けている間、Click イベントを連続発火するボタン。
  • Menu — コマンドを階層メニュー形式で提供する代替手段。
