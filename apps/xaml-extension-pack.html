Xaml.ExtensionPack

A lightweight, dependency-free XAML extension library for WPF. Provides essential MVVM foundations and WPF-specific attached behaviors as NuGet packages.

Overview

Xaml.ExtensionPack is a set of NuGet libraries designed to eliminate boilerplate in WPF/XAML application development. It is split into two packages to keep dependencies minimal:

  • Xaml.ExtensionPack — A framework-agnostic core library providing foundational MVVM logic. Targets .NET 10, .NET 8, and .NET Framework 4.7.2.
  • Xaml.ExtensionPack.Wpf — A WPF-specific extension built on top of the core package, providing attached behaviors and UI-focused properties. Targets .NET 10 (Windows), .NET 8 (Windows), and .NET Framework 4.7.2.

Background: Cutting Boilerplate Without Heavyweight Dependencies

WPF applications built with the MVVM pattern tend to accumulate repetitive infrastructure code: INotifyPropertyChanged implementations, ICommand wrappers, and async-safe fire-and-forget helpers. While community toolkits exist, they often pull in large dependency trees or enforce architectural conventions beyond what a project needs.

Xaml.ExtensionPack was created to provide only what is genuinely necessary — a small, focused set of utilities that integrate cleanly into any WPF project without extra overhead. The split into a core package and a WPF-specific package ensures that non-WPF projects can also benefit from the MVVM utilities.

What's Included

Xaml.ExtensionPack (Core)

  • BindableBase — Abstract base class implementing INotifyPropertyChanged. Provides SetProperty helpers (with optional callback) and lambda-based OnPropertyChanged for refactoring-safe notifications.
  • RelayCommand / RelayCommand<T> — Lightweight ICommand implementations that delegate execution logic via delegates. Support optional CanExecute predicates and expose RaiseCanExecuteChanged via the IRaiseCanExecuteChanged interface.
  • AsyncRelayCommand — An async-aware ICommand that prevents concurrent execution and automatically updates CanExecute while a task is running.
  • TaskExtensions.FireAndForget — A safe fire-and-forget extension method for Task that accepts an optional exception handler, avoiding unobserved task exceptions.

Xaml.ExtensionPack.Wpf

  • WPF-specific attached behaviors and UI-focused properties — Extends the core package with behaviors designed for WPF controls, enabling clean XAML-driven interaction patterns without code-behind.
