MessageBoxOptions — Display Options

A ComboBox control for selecting special display and behavior options for the MessageBox. These options control text alignment, reading direction, and desktop targeting.

← Back to WPF MessageBox Sample Creator

Control: ComboBox

The ComboBox is a standard WPF dropdown control that lets users select one value from a list. In this app, the MessageBoxOptions field uses a ComboBox configured with the following properties:

  • IsEditable="False" — Selection is limited to predefined enum values.
  • ItemsSource="{Binding Source={StaticResource MessageBoxOptionsDataSource}}" — Automatically populated with all values of the MessageBoxOptions enum.
  • SelectedIndex="1" — Defaults to the second item on startup.
  • SelectedItem="{Binding MessageBoxOptions}" — The selected enum value is bound to the ViewModel.

Property: MessageBoxOptions

The MessageBoxOptions enum is the sixth argument of MessageBox.Show(). It provides additional flags to control how the MessageBox is displayed.

Values and Behavior

None (value: 0)

No special options applied. The MessageBox is displayed with standard left-aligned text on the active desktop. This is the default setting for most scenarios.

DefaultDesktopOnly (value: 131072)

Forces the MessageBox to appear on the current active (default) desktop only. This is useful for services or background processes to ensure the dialog is visible on the user's active screen. It prevents the dialog from appearing on secure desktops (such as the Windows lock screen or UAC prompt desktop).

RightAlign (value: 524288)

Right-aligns the message text within the dialog. The text content is displayed aligned to the right margin instead of the default left alignment. Useful for languages that are conventionally right-aligned.

RtlReading (value: 1048576)

Displays the message text in right-to-left reading order. This is designed for right-to-left languages such as Arabic and Hebrew. The layout of the dialog mirrors horizontally: text flows right-to-left, and the icon appears on the right side.

ServiceNotification (value: 2097152)

Intended for use in Windows services or background processes that run without a parent window. When this option is set, the MessageBox is displayed on the active desktop even if no user is currently logged on. This option should not be used from a regular interactive application.

Code Reference

In the app, the selected value is passed as the sixth argument to MessageBox.Show(): 

MessageBox.Show(
    ViewModel.MessageBoxText,
    ViewModel.Caption,
    ViewModel.MessageBoxButton,
    ViewModel.MessageBoxImage,
    ViewModel.MessageBoxResult,
    ViewModel.MessageBoxOptions  // ← This ComboBox value
);
