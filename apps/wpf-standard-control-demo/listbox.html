ListBox List

ListBox is a control that displays a list of items from which users can select one or multiple items. It supports single, multiple, and extended selection modes, and can be bound to data collections.

Overview

The WPF ListBox control presents a scrollable list of selectable items and inherits from Selector, which itself extends ItemsControl. This means ListBox participates in WPF's full data-binding infrastructure: you can bind its ItemsSource to any IEnumerable, define a DataTemplate for rich visual item rendering, and use DisplayMemberPath for simple string property display without a template.

Selection is one of ListBox's key differentiators from ComboBox. The SelectionMode property offers three options: Single for single-item selection (the default), Multiple for toggling individual items independently with a single click, and Extended for contiguous range selection with Shift+click and Ctrl+click for non-contiguous multi-selection. This flexibility makes ListBox suitable for both simple pickers and complex multi-select scenarios.

The SelectedIndex, SelectedItem, and SelectedValue properties all provide access to the current selection from different angles. SelectedIndex gives the zero-based position; SelectedItem gives the actual data object; and SelectedValue, combined with SelectedValuePath, extracts a specific property from the selected item — for example, binding only the Id property of a complex object without exposing the entire object to the view model.

ListBox wraps its items in a ScrollViewer, and the attached ScrollViewer.HorizontalScrollBarVisibility and ScrollViewer.VerticalScrollBarVisibility properties let you control scroll bar visibility without subclassing. Setting these to Disabled removes the scrollbars entirely, Auto shows them only when needed, and Visible always shows them.

Demonstrated Properties

The following properties are demonstrated interactively in the WPF Standard Control Demo App. Each property can be configured in real time within the app to observe its behavior.

PropertyValuesDescription
SelectionMode Single / Multiple / Extended Determines how many items can be selected and how selection interactions work. Extended is the richest mode, supporting Shift-click for range selection and Ctrl-click for non-contiguous selection, matching the behavior of file explorer list views.
DisplayMemberPath (ItemsControl) string The name of a property on the data item to display as the item's text. When set, ListBox automatically renders each item using that property value as a TextBlock, eliminating the need for a DataTemplate for simple string-display scenarios.
SelectedIndex (Selector) int The zero-based index of the currently selected item, or -1 when no item is selected. Two-way binding on this property allows the view model to both read the current selection and programmatically select an item by its list position.
SelectedItem (Selector) object The actual data object that is currently selected in the list. This is the most MVVM-friendly binding target, as it directly exposes the domain model object rather than requiring index-based lookups in code.
SelectedValuePath / SelectedValue (Selector) string / object SelectedValuePath names a property on the data item (e.g., "Id") and SelectedValue reflects only that property from the selected item. This is useful for binding a foreign key or identifier without needing the full object reference.
IsSelected (ListBoxItem) bool An item-level property on each ListBoxItem container that indicates whether that specific item is selected. It can be used in item data templates within triggers to change item appearance based on selection state.
HorizontalScrollBarVisibility / VerticalScrollBarVisibility (ScrollViewer) Disabled / Auto / Hidden / Visible Attached properties from ScrollViewer that control scroll bar display. Auto is the typical choice — scrollbars appear only when content overflows — while Disabled completely prevents scrolling for fixed-height lists.

XAML Example

The following XAML demonstrates a data-bound ListBox with a custom DataTemplate and extended multi-selection:

<ListBox ItemsSource="{Binding Employees}"
         SelectionMode="Extended"
         SelectedItem="{Binding SelectedEmployee}"
         ScrollViewer.VerticalScrollBarVisibility="Auto"
         Margin="8">
  <ListBox.ItemTemplate>
    <DataTemplate>
      <StackPanel Orientation="Horizontal" Margin="4">
        <TextBlock Text="{Binding Name}"
                   FontWeight="SemiBold"
                   Width="140" />
        <TextBlock Text="{Binding Department}"
                   Foreground="#555"
                   Margin="8,0,0,0" />
      </StackPanel>
    </DataTemplate>
  </ListBox.ItemTemplate>
</ListBox>

Common Use Cases

  • Settings option pickers: Displaying a list of configuration options where the user selects one or more items, such as a list of enabled plugins or export formats.
  • File selection panels: Showing a list of files or records from which the user selects one or more for a batch operation like delete, export, or process.
  • Navigation panels: Sidebar lists that show categories, tags, or sections, where selecting an item updates the main content area.
  • Transfer list UIs: Dual-ListBox patterns where items are moved between "Available" and "Selected" lists using Add/Remove buttons.
  • Filtered data views: Binding ListBox to a CollectionView with filtering so the list dynamically narrows based on a search term entered in a linked TextBox.

Tips and Best Practices

  • Use VirtualizingStackPanel for large lists: By default, ListBox already uses a virtualizing panel, so only visible items are rendered. Avoid replacing it with a non-virtualizing panel for collections with hundreds of items.
  • Bind SelectedItem two-way: Setting Mode=TwoWay on the SelectedItem binding ensures that programmatic selection from the view model is reflected in the UI without additional event handling.
  • Use ObservableCollection as ItemsSource: Collections implementing INotifyCollectionChanged automatically keep the ListBox in sync when items are added or removed.
  • Define keyboard shortcuts for selection: For Extended mode, document Ctrl+A (select all) in the UI so users know it is available — it works by default but is not obvious.
  • Consider ListView for tabular data: If items have multiple columns, switch to ListView with GridView for a proper columnar display with resizable headers.

Related Controls

  • ListView — extends ListBox with a pluggable view model, most commonly used for tabular GridView layouts.
  • ComboBox — a compact dropdown list for single-item selection where screen space is limited.
  • DataGrid — a fully featured grid for tabular data with inline editing, sorting, and grouping.
