DataGrid List

DataGrid is a powerful control for displaying tabular data with features like sorting, filtering, grouping, column resizing, and in-place editing. It automatically generates columns from data source properties or allows custom column definitions.

Overview

The WPF DataGrid is the most feature-rich data-presentation control in the WPF standard library. It renders a two-dimensional table of rows and columns, directly analogous to a spreadsheet or database table view. Columns can be auto-generated from the bound data source's property reflection, or defined explicitly in XAML using typed column classes: DataGridTextColumn, DataGridCheckBoxColumn, DataGridComboBoxColumn, DataGridHyperlinkColumn, and the highly flexible DataGridTemplateColumn for fully custom cell content.

Editing is built in: double-clicking or pressing F2 puts a cell into edit mode, where the appropriate editor for the column type is shown. DataGridTextColumn shows a TextBox; DataGridCheckBoxColumn shows a CheckBox; and DataGridComboBoxColumn shows a ComboBox. Changes are committed on focus loss or Enter, and cancelled with Escape. Row-level validation is supported through IDataErrorInfo or INotifyDataErrorInfo, and error indicators appear automatically in the row header.

Sorting is enabled by default for auto-generated columns and for explicitly defined columns when CanUserSortColumns="True". Clicking a column header sorts ascending; clicking again sorts descending; a third click reverts to unsorted. Grouping and filtering require a CollectionViewSource wrapper in the ViewModel — the DataGrid renders group headers automatically when the source view has group descriptions applied.

Performance is a key consideration for large datasets. Enable EnableRowVirtualization="True" (the default) and EnableColumnVirtualization="True" so that only visible cells are realized in the visual tree. Frozen columns (FrozenColumnCount) allow a fixed number of leading columns to remain visible during horizontal scrolling, which is essential for usability in wide grids.

Demonstrated Properties

The following properties are demonstrated interactively in the WPF Standard Control Demo App. Each property can be configured in real time within the app to observe its behavior.

Property Values Description
AutoGenerateColumns bool When True, WPF inspects the bound data source via reflection and creates a column for each public property. This is the fastest way to display a data object but provides no control over column order, width, or cell templates — use explicit column definitions for production UIs.
IsReadOnly bool Prevents all in-place editing across the entire grid when set to True. Individual columns can independently set IsReadOnly even when the grid is editable, allowing mixed read/write grids where certain columns (e.g., IDs) are display-only.
SelectionMode / SelectionUnit Single/Extended / Cell/FullRow/CellOrRowHeader SelectionMode controls whether one or multiple items can be selected; SelectionUnit controls the granularity — cell-level or full-row. Combining Extended and FullRow replicates the behaviour of a traditional multi-select list.
ColumnWidth / MaxColumnWidth / MinColumnWidth DataGridLength / double / double Sets the default width for all columns as a DataGridLength value (pixel, star, or auto), while MaxColumnWidth and MinColumnWidth clamp resizing. DataGridLength.SizeToCells auto-sizes to the widest cell value.
CanUserAddRows / CanUserDeleteRows bool Enables or disables the new-row placeholder at the bottom of the grid and the Delete key row removal. Both require the bound collection to implement IEditableCollectionView (e.g., ObservableCollection<T> wrapped in a ListCollectionView).
CanUserReorderColumns / CanUserResizeColumns / CanUserResizeRows / CanUserSortColumns bool Fine-grained user-interaction flags. Disable column reordering or resizing to enforce a fixed layout, or disable sorting if the data has a meaningful intrinsic order that should not be disturbed by the user.
RowDetailsVisibilityMode / AreRowDetailsFrozen Collapsed/Visible/VisibleWhenSelected / bool RowDetailsVisibilityMode controls whether a row detail template is shown below each row. VisibleWhenSelected is the most common choice, expanding the selected row to show additional information defined in RowDetailsTemplate.
GridLinesVisibility / HorizontalGridLinesBrush / VerticalGridLinesBrush None/Horizontal/Vertical/All / Brush / Brush Controls which grid lines are drawn and their colours. Setting GridLinesVisibility="None" and using alternating row backgrounds creates a modern, borderless table look popular in flat design applications.
AlternatingRowBackground Brush Sets the background brush for even-numbered rows (index 0, 2, 4…), creating a zebra-stripe effect that helps the eye track across wide rows. Combine with a contrasting RowBackground for odd rows.
FrozenColumnCount int Specifies how many leading columns remain stationary during horizontal scrolling. Setting this to 1 or 2 for an identifier column (Name, ID) makes wide grids much easier to navigate without losing context of which row you are editing.
HeadersVisibility None/Column/Row/All Controls which header areas are shown. Hiding the row header (Column only) saves horizontal space when row-level features like drag-reorder or row selection indicators are not needed.
EnableRowVirtualization / EnableColumnVirtualization bool Enables UI virtualisation for rows and columns respectively, significantly reducing memory usage and render time for large datasets. Column virtualisation should be enabled cautiously as it can cause slight rendering artefacts on horizontal scroll in some WPF versions.
RowBackground / RowHeight / MinRowHeight Brush / double / double Sets the default row background brush and the height of each row. MinRowHeight prevents rows from collapsing to zero when content is empty, maintaining visual consistency.
RowStyle / CellStyle / ColumnHeaderStyle / ColumnHeaderTemplate Style / Style / Style / DataTemplate Apply uniform styling to all rows, cells, or column headers via Style resources. Use ColumnHeaderTemplate to embed custom header content such as a filter TextBox directly inside the column header area.

XAML Example

The following XAML defines a DataGrid with explicit columns, alternating row colours, and frozen column:

<DataGrid ItemsSource="{Binding Employees}"
          AutoGenerateColumns="False"
          IsReadOnly="False"
          SelectionMode="Extended"
          SelectionUnit="FullRow"
          FrozenColumnCount="1"
          AlternatingRowBackground="#F5F5F5"
          GridLinesVisibility="Horizontal"
          CanUserAddRows="False"
          CanUserDeleteRows="True"
          EnableRowVirtualization="True">
  <DataGrid.Columns>
    <DataGridTextColumn Header="Name"
                        Binding="{Binding Name}"
                        Width="160"
                        IsReadOnly="True" />
    <DataGridTextColumn Header="Department"
                        Binding="{Binding Department}"
                        Width="140" />
    <DataGridCheckBoxColumn Header="Active"
                            Binding="{Binding IsActive}"
                            Width="60" />
    <DataGridTemplateColumn Header="Actions" Width="80">
      <DataGridTemplateColumn.CellTemplate>
        <DataTemplate>
          <Button Content="Edit"
                  Command="{Binding DataContext.EditCommand,
                    RelativeSource={RelativeSource AncestorType=DataGrid}}"
                  CommandParameter="{Binding}" />
        </DataTemplate>
      </DataGridTemplateColumn.CellTemplate>
    </DataGridTemplateColumn>
  </DataGrid.Columns>
</DataGrid>

Common Use Cases

  • CRUD data management screens: Employee lists, product catalogues, or order management views where users view, add, edit, and delete records in-place.
  • Financial and reporting tables: Read-only summary grids with alternating row colours, frozen identifier columns, and custom number formatting in cell templates.
  • Configuration editors: Property-grid-style tables where rows represent settings and values are edited inline, often with a DataGridComboBoxColumn for enumerated options.
  • Log and event viewers: Read-only, auto-scroll grids displaying real-time event streams, leveraging row virtualisation to handle thousands of entries without performance degradation.
  • Expandable row details: Master-detail layouts using RowDetailsTemplate to show related child data below the selected row, avoiding a separate detail panel.

Tips and Best Practices

  • Always define columns explicitly in production applications. AutoGenerateColumns exposes all public properties including internal ones, and column order depends on reflection order, which is not guaranteed.
  • Enable virtualisation (EnableRowVirtualization="True") for any list that could contain more than a few dozen rows. Without it, the DataGrid instantiates visual elements for every row at load, causing sluggish startup for large datasets.
  • Use ObservableCollection<T> as the ItemsSource so the grid automatically reflects adds and removes without a full refresh. For sorted or filtered views, wrap it in a CollectionViewSource.
  • Freeze identifier columns using FrozenColumnCount when your grid has more columns than fit in the viewport. This simple setting dramatically improves horizontal-scroll usability.
  • Validate with INotifyDataErrorInfo rather than exception-based validation. It provides per-property error messages that the DataGrid can display in tooltips without crashing the edit session.
