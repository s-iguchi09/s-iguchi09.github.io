ComboBox Inputs

ComboBox is a dropdown list control that allows users to select an item from a list. It can also be made editable to allow users to type custom values. It supports both simple item lists and complex data-bound collections.

Overview

The WPF ComboBox combines a selection display area with a collapsible dropdown list, making it the standard control for choosing one value from a discrete set while conserving screen space. In its default (non-editable) form, it behaves like a traditional drop-down: the user clicks the control to reveal the list, selects an item, and the list closes. The selected item is then displayed in the collapsed state.

Setting IsEditable="True" transforms the ComboBox into an editable text field backed by the dropdown list. This enables auto-complete behaviour — as the user types, WPF filters or scrolls to the closest matching item. IsReadOnly can further restrict text input while still showing the dropdown, useful for cases where you want the visual appearance of a text box but only allow pre-defined choices.

For data-driven scenarios, ComboBox inherits from ItemsControl, which means its items can be populated via ItemsSource binding to any collection. The DisplayMemberPath property selects which property of a complex object is displayed as the item label, while SelectedValuePath determines which property is exposed through SelectedValue. This pattern is essential for database-backed foreign-key selectors such as country pickers or category selectors.

The IsSynchronizedWithCurrentItem property integrates the ComboBox with WPF's CollectionView currency mechanism. When enabled, selecting an item in the ComboBox also advances the current item in a shared CollectionViewSource, which allows multiple bound controls — such as a ComboBox and a detail form — to stay automatically synchronized without extra event wiring.

Demonstrated Properties

The following properties are demonstrated interactively in the WPF Standard Control Demo App. Each property can be configured in real time within the app to observe its behavior.

Property Values Description
IsDropDownOpen bool Gets or sets whether the dropdown list is currently visible. You can bind or set this programmatically to open or close the list from code, for example to show the list automatically when a dialog opens.
IsEditable bool When True, replaces the read-only selection display with an editable text box, allowing the user to type values directly. This enables type-ahead/auto-complete behaviour and is required before IsReadOnly has any effect.
IsReadOnly bool Effective only when IsEditable is True. Prevents the user from typing in the text field while still allowing dropdown selection, giving you a non-editable combobox that visually looks like a text box.
MaxDropDownHeight double Limits the maximum pixel height of the open dropdown list. When the list contains more items than fit in this height, a scrollbar appears automatically. Setting a reasonable value prevents the dropdown from obscuring large portions of the screen.
Text string Represents the current text in the editable portion of the ComboBox when IsEditable is True. Binding this property two-way allows you to read user input and also programmatically set the displayed text without changing the selection.
StaysOpenOnEdit bool Controls whether the dropdown list remains open while the user is typing (requires IsEditable="True"). When True, the list stays visible during text entry, enabling inline filtering of options as the user types.
ShouldPreserveUserEnteredPrefix bool Determines whether the text the user has typed is preserved as they navigate the dropdown with the keyboard. When True, the typed prefix is kept, aiding auto-complete scenarios where partial input guides list navigation.
SelectedIndex (Selector) int Gets or sets the zero-based index of the currently selected item. A value of -1 means no item is selected. This property is useful for programmatically selecting the first item on load or resetting the selection.
SelectedItem (Selector) object Gets or sets the selected item object directly. When the ItemsSource is a collection of complex objects, binding SelectedItem two-way to a ViewModel property gives you the full item object on selection, not just the display string.
SelectedValue / SelectedValuePath / DisplayMemberPath (ItemsControl) object / string / string SelectedValuePath names the property of the data object to expose as SelectedValue. DisplayMemberPath names the property shown in the list. This pair lets you bind a foreign key integer (SelectedValue) while displaying a friendly name to the user.
IsSynchronizedWithCurrentItem (ItemsControl) bool? When True, the ComboBox selection is kept in sync with the current item of the underlying CollectionView. This is the standard way to create master-detail views where selecting a ComboBox item automatically updates a bound details panel.

XAML Example

The following XAML demonstrates data binding patterns and an editable ComboBox configuration:

<StackPanel Margin="16" Spacing="12">

  <!-- Simple string list -->
  <ComboBox Width="200" SelectedIndex="0">
    <ComboBoxItem>Option A</ComboBoxItem>
    <ComboBoxItem>Option B</ComboBoxItem>
    <ComboBoxItem>Option C</ComboBoxItem>
  </ComboBox>

  <!-- Data-bound with display and value paths -->
  <ComboBox Width="220"
            ItemsSource="{Binding Countries}"
            DisplayMemberPath="Name"
            SelectedValuePath="Id"
            SelectedValue="{Binding SelectedCountryId}" />

  <!-- Editable with type-ahead and capped dropdown height -->
  <ComboBox Width="220"
            IsEditable="True"
            StaysOpenOnEdit="True"
            MaxDropDownHeight="160"
            ItemsSource="{Binding Cities}"
            Text="{Binding CityText, UpdateSourceTrigger=PropertyChanged}" />

</StackPanel>

Common Use Cases

  • Country / region pickers: Binding a ComboBox to a lookup collection with DisplayMemberPath="Name" and SelectedValuePath="Code" to select and store ISO country codes.
  • Enumeration selectors: Exposing a set of enum values (priority levels, status codes, sort orders) in a dropdown, optionally using a value converter for display-friendly strings.
  • Search-and-select: Editable ComboBox with server-side suggestions loaded dynamically as the user types, simulating an auto-complete search box.
  • Unit and currency selectors: Compact dropdowns in data-entry forms where the user picks the unit (kg, lb, cm) alongside a numeric input field.
  • Theme / language pickers: Application-level settings dropdowns that immediately apply a new resource dictionary or locale when the selection changes.

Tips and Best Practices

  • Prefer SelectedItem binding over SelectedIndex when working with data-bound collections. Index-based selection breaks when the collection is sorted or filtered.
  • Use SelectedValuePath for foreign keys. Rather than binding SelectedItem to a full entity object, bind SelectedValue to a simple ID property to keep ViewModel dependencies minimal.
  • Set MaxDropDownHeight for large collections. Without a height limit, a dropdown with hundreds of items can extend off-screen, degrading the user experience.
  • Provide an empty/placeholder item (e.g., "— Select —" at index 0) rather than leaving the combobox blank at startup. This makes it clear to the user that a selection is required.
  • Use ItemTemplate for complex items. If list items need icons or multi-line content, define a DataTemplate as ItemTemplate rather than relying solely on DisplayMemberPath.

Related Controls

  • ListView — shows all items at once without collapsing; suited for longer lists that benefit from always-visible selection.
  • RadioButton — mutually exclusive selection directly visible in the UI; prefer for small option sets (2–5 items).
  • TextBox — free-text input without a predefined list; use when no suggestion set exists.
  • TreeView — hierarchical selection; use when choices have a parent-child structure.
