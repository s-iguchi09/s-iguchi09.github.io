TextBox Inputs

TextBox is the primary editable text input control in WPF. It supports single-line and multiline input, text wrapping, selection, undo/redo, and keyboard/mouse text manipulation. It is one of the most fundamental and frequently used controls in any data-entry form.

Overview

The WPF TextBox inherits from TextBoxBase, which provides the core editing engine including selection, clipboard operations (Cut, Copy, Paste), undo/redo history, and programmatic scroll methods. TextBox adds single-line display features such as CharacterCasing, MaxLength, and TextAlignment on top of the base class.

For MVVM data binding, the Text property is the primary binding target. Two-way binding with UpdateSourceTrigger=PropertyChanged pushes every keystroke to the ViewModel, enabling live validation and real-time UI updates. The default UpdateSourceTrigger=LostFocus updates the ViewModel only when the TextBox loses focus, which reduces unnecessary updates but delays validation feedback.

Multiline input is enabled by setting TextWrapping="Wrap" and optionally setting MinLines and MaxLines to control the visible height. Combined with AcceptsReturn="True" (inherited from TextBoxBase), this creates a full text area suitable for notes, comments, and code editors.

CharacterCasing provides automatic case normalization — setting it to Upper ensures all typed characters are stored as uppercase regardless of the shift key state, useful for postal codes, product codes, and other fixed-case identifiers. Lower enforces lowercase input similarly.

Demonstrated Properties

The following properties are demonstrated interactively in the WPF Standard Control Demo App. Each property can be configured in real time within the app to observe its behavior.

PropertyValuesDescription
Text string The string value currently entered in the TextBox. Bind two-way to a ViewModel string property for standard data-entry use. Use UpdateSourceTrigger=PropertyChanged for live validation or LostFocus for batched updates on tab-out.
TextWrapping NoWrap / Wrap / WrapWithOverflow Controls whether long text wraps to new lines. Wrap is needed for multiline text areas; NoWrap (default) keeps single-line behavior with horizontal scrolling. WrapWithOverflow allows individual unbreakable tokens (e.g., long URLs) to overflow rather than clip.
TextDecorations None / Underline / Strikethrough / OverLine / Baseline Applies text decoration to all text in the box. Commonly used to show Underline for link-style inputs or Strikethrough for deprecation indicators. Multiple decorations can be combined.
TextAlignment Left / Right / Center / Justify Aligns the text horizontally within the TextBox. Right is useful for numeric inputs; Center for title-style single-line fields; Justify for paragraph areas.
MaxLength int (0 = unlimited) Limits how many characters the user can type. The TextBox silently rejects input beyond this limit. Setting 0 (default) means unlimited. Use this for fields with fixed-length constraints such as postal codes, phone numbers, or PINs.
MaxLines int Limits the maximum number of visible lines in a multiline TextBox. Additional lines can be scrolled. A common pattern is to set MinLines="3" MaxLines="10" for a textarea that grows up to ten rows.
MinLines int Sets the minimum height of the TextBox expressed in lines. Even with no content, the control reserves space for at least this many lines, preventing jarring resize when text is added. Useful for comment fields that should always appear as a textarea.
CharacterCasing Normal / Upper / Lower Automatically converts all typed input to the specified case. Upper forces uppercase — typed "hello" appears as "HELLO" in both the display and the bound Text property. Useful for license keys, locale codes, and identifiers that must be uppercase.

XAML Example

Various TextBox configurations for common form patterns:

<!-- Standard single-line input with live validation -->
<TextBox Text="{Binding Username, Mode=TwoWay,
               UpdateSourceTrigger=PropertyChanged}"
         MaxLength="50" />

<!-- Multiline comment area -->
<TextBox Text="{Binding Notes, Mode=TwoWay}"
         TextWrapping="Wrap" AcceptsReturn="True"
         MinLines="3" MaxLines="8"
         VerticalScrollBarVisibility="Auto" />

<!-- Uppercase postal code field -->
<TextBox Text="{Binding PostalCode, Mode=TwoWay}"
         CharacterCasing="Upper"
         MaxLength="10"
         TextAlignment="Center"
         Width="100" />

Common Use Cases

  • Single-line text input — Name, email, username, and search fields in forms and dialogs.
  • Multiline text areas — Notes, comments, address fields, and code snippet editors.
  • Numeric input with formatting — Currency or quantity fields using TextAlignment="Right" combined with a converter for formatted display.
  • Fixed-format fields — License keys, postal codes, and phone numbers using MaxLength and CharacterCasing.
  • Search boxes — Live-filter a collection by binding Text with UpdateSourceTrigger=PropertyChanged to a search string ViewModel property.

Tips and Best Practices

  • Use UpdateSourceTrigger wisely — PropertyChanged enables live validation but triggers frequently; LostFocus is less intrusive for simple forms without live feedback needs.
  • Add a watermark with placeholder text — WPF's TextBox does not natively support placeholder text, but you can overlay a TextBlock bound to Text with a converter, or use an attached behavior.
  • SpellCheck.IsEnabled — Set SpellCheck.IsEnabled="True" on any TextBox to enable built-in red-underline spell checking without any extra code.
  • Use PasswordBox for secrets — Never use a TextBox for passwords; the text is unmasked. Use PasswordBox instead.
  • Validation integration — Implement INotifyDataErrorInfo in the ViewModel and set ValidatesOnNotifyDataErrors=True on the binding to show red borders and error tooltips automatically.

Related Controls

  • PasswordBox — For masked password input; does not expose the text as a bindable string by default.
  • TextBlock — The read-only counterpart for displaying text.
  • ComboBox — Provides editable text input combined with a dropdown suggestion list.
