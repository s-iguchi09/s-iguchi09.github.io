ListView List

ListView is an advanced list control that extends ListBox with a pluggable view model. The most common view is GridView, which displays items in a tabular format with column headers that support resizing and reordering.

Overview

The WPF ListView control builds upon ListBox by introducing the View property, which accepts a ViewBase object that defines how items are rendered. The built-in GridView view renders items as a table with named column headers, making ListView the go-to control for read-only data grids or simple file-explorer-style lists. Custom views can also be implemented by subclassing ViewBase, enabling tile views, card views, or any other presentation format.

When using GridView, columns are defined through the Columns property as a collection of GridViewColumn objects. Each column specifies a Header for the column title, a Width, and either a DisplayMemberBinding pointing to a data property or a CellTemplate for custom cell rendering. The AllowsColumnReorder property enables drag-and-drop column reordering, and columns can be resized at runtime by dragging column dividers.

ListView inherits all selection features from ListBox, including the SelectionMode property with Single, Multiple, and Extended options. The SelectedItem and SelectedIndex bindings work identically to ListBox, making it straightforward to migrate between the two controls as requirements evolve. The IsSelected property on ListViewItem containers supports selection-aware styling in item templates.

For large datasets, ListView combined with VirtualizingStackPanel and ScrollViewer provides the performance characteristics needed to display thousands of rows without excessive memory consumption. The ScrollViewer.HorizontalScrollBarVisibility and ScrollViewer.VerticalScrollBarVisibility attached properties fine-tune scroll bar behavior to match the target layout requirements.

Demonstrated Properties

The following properties are demonstrated interactively in the WPF Standard Control Demo App. Each property can be configured in real time within the app to observe its behavior.

PropertyValuesDescription
SelectionMode (ListBox) Single / Multiple / Extended Inherited from ListBox, this controls how many items can be selected simultaneously. Extended mode with Shift- and Ctrl-click is commonly used in file-manager-style ListViews where multi-file operations are needed.
View ViewBase The pluggable view object that defines how list items are rendered. Setting this to a GridView instance activates tabular column display; setting it to null falls back to a plain vertical list similar to ListBox.
Columns (GridView) GridViewColumnCollection The collection of GridViewColumn objects that define each column's header, width, and cell binding. Columns can be added or removed at runtime to implement show/hide column features, and they can be reordered by the user when AllowsColumnReorder is enabled.
AllowsColumnReorder (GridView) bool When true, users can drag column headers to reorder the columns. This interactive feature is expected behavior in file-manager and data-grid UIs and significantly improves usability without requiring any code-behind logic.
Width (GridViewColumn) double The width of an individual column in device-independent pixels. Setting Width="NaN" (the default when omitted) makes the column auto-size to its content, while a fixed pixel value creates a stable column layout across different data.
SelectedIndex / SelectedItem (Selector) int / object Zero-based index and data object of the currently selected item. These work identically to ListBox, and two-way binding on SelectedItem is the recommended MVVM pattern for responding to and programmatically controlling selection.
IsSelected (ListBoxItem) bool An item container property that can be used in DataTrigger or Style setters to apply row-level visual changes (e.g., bold text, highlight color) to selected rows within the ListView.
HorizontalScrollBarVisibility / VerticalScrollBarVisibility (ScrollViewer) Disabled / Auto / Hidden / Visible Attached properties that control the ListView's built-in scroll bars. For a table with many columns, enabling HorizontalScrollBarVisibility="Auto" ensures columns are accessible even when the window is narrow.

XAML Example

The following XAML demonstrates a ListView with GridView columns, column reordering enabled, and selection binding:

<ListView ItemsSource="{Binding Orders}"
          SelectedItem="{Binding SelectedOrder}"
          SelectionMode="Single"
          ScrollViewer.HorizontalScrollBarVisibility="Auto"
          Margin="8">
  <ListView.View>
    <GridView AllowsColumnReorder="True">
      <GridViewColumn Header="Order ID"
                      DisplayMemberBinding="{Binding OrderId}"
                      Width="80" />
      <GridViewColumn Header="Customer"
                      DisplayMemberBinding="{Binding CustomerName}"
                      Width="160" />
      <GridViewColumn Header="Date"
                      DisplayMemberBinding="{Binding OrderDate, StringFormat=d}"
                      Width="100" />
      <GridViewColumn Header="Total"
                      DisplayMemberBinding="{Binding Total, StringFormat=C}"
                      Width="90" />
      <GridViewColumn Header="Status" Width="100">
        <GridViewColumn.CellTemplate>
          <DataTemplate>
            <TextBlock Text="{Binding Status}"
                       Foreground="{Binding Status,
                         Converter={StaticResource StatusColorConverter}}" />
          </DataTemplate>
        </GridViewColumn.CellTemplate>
      </GridViewColumn>
    </GridView>
  </ListView.View>
</ListView>

Common Use Cases

  • File and folder browsers: Displaying files with Name, Size, Type, and Modified Date columns, with column reordering and resizing for user customization.
  • Order and invoice lists: Business applications listing transactional records with sortable columns for date, amount, status, and customer information.
  • Log viewers: Presenting application log entries in timestamped columns, often with color coding based on severity level using cell templates.
  • Search results panels: Showing search hits in a list with columns for relevance score, title, source, and date, allowing users to sort and select results.
  • Report tables: Lightweight read-only data tables in dashboard panels where full DataGrid editing features are not needed.

Tips and Best Practices

  • Use GridViewColumn.CellTemplate for rich cells: For columns requiring icons, badges, or formatted content beyond plain text, define a DataTemplate in CellTemplate instead of relying on DisplayMemberBinding.
  • Implement column sorting manually: Unlike DataGrid, ListView/GridView does not sort automatically on header click. Handle the GridViewColumnHeader.Click event and apply a SortDescription to the CollectionView.
  • Size columns with Auto for initial fit: Setting Width="Auto" on columns sizes them to their content on first load, but users can still resize them manually if AllowsColumnReorder is enabled.
  • Use VirtualizingPanel.IsVirtualizing="True": This is on by default but can be accidentally disabled. Verify virtualization is active when binding to large collections to maintain scrolling performance.
  • Prefer ListView over DataGrid for read-only tables: If you don't need inline editing, sorting indicators, or row grouping, ListView is lighter and easier to style than the full DataGrid.

Related Controls

  • ListBox — ListView's base class; use when a simple single-column list without headers is sufficient.
  • DataGrid — a full-featured grid with built-in sorting, filtering, grouping, and inline editing.
  • GridView — the view object used inside ListView to define tabular column layout.
