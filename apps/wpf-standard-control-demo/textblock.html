TextBlock Display

TextBlock is a lightweight, non-interactive control for displaying text. It is the primary read-only text element in WPF, supporting rich inline formatting, text trimming, wrapping, and flow document elements. Unlike Label, it does not participate in tab focus and has no border or padding by default.

Overview

The WPF TextBlock is the go-to control for displaying static or data-bound text in the UI. It derives from FrameworkElement directly (not from Control), which gives it a smaller visual footprint than label-based controls. This makes it the recommended choice for high-density UIs such as list item templates and data grids where thousands of text elements may be created.

TextBlock supports rich inline text formatting through the Inlines collection. By mixing Run, Bold, Italic, Underline, Hyperlink, and LineBreak inline elements, you can produce formatted paragraphs in a single TextBlock without using a full FlowDocument. Inline content and the Text property are mutually exclusive — when you add inline elements, the Text property reflects the concatenated plain text.

The TextTrimming property adds ellipsis characters when text overflows its bounds, giving users a visual cue that content has been cut. CharacterEllipsis trims mid-word, while WordEllipsis trims at a word boundary for a cleaner result. Combined with a fixed width and a ToolTip showing the full text, this pattern is ubiquitous in list and grid cells.

The LineHeight and LineStackingStrategy properties give fine-grained control over the vertical spacing of lines in a multi-line TextBlock. Setting a custom LineHeight overrides the font's natural leading; LineStackingStrategy determines whether the height is applied to the block's lines uniformly (BlockLineHeight) or based on each line's maximum glyph height (MaxHeight).

Demonstrated Properties

The following properties are demonstrated interactively in the WPF Standard Control Demo App. Each property can be configured in real time within the app to observe its behavior.

PropertyValuesDescription
Text string The plain text content to display. Data-bind this to a string property in the ViewModel for dynamic text. When using inline elements, Text returns the concatenated plain-text equivalent.
TextWrapping NoWrap / Wrap / WrapWithOverflow Wrap breaks long text across multiple lines at word boundaries; WrapWithOverflow allows individual words that exceed the width to overflow rather than being clipped; NoWrap (default) keeps all text on one line.
TextTrimming None / CharacterEllipsis / WordEllipsis Controls how overflowing text is trimmed. CharacterEllipsis cuts at the character level and appends "…"; WordEllipsis cuts at a word boundary for a cleaner ellipsis. Pair with a ToolTip binding to show the full text on hover.
TextAlignment Left / Right / Center / Justify Sets the horizontal alignment of text within the TextBlock. Justify distributes whitespace to align both left and right edges, similar to newspaper columns. Useful for paragraph-style content blocks.
Padding Thickness Adds internal spacing between the TextBlock boundary and the text content. Useful when using Background to create a colored label effect without nesting inside a Border element.
FontWeight Thin / ExtraLight / Light / Normal / Medium / SemiBold / Bold / ExtraBold / Black Sets the weight (thickness) of the font strokes. Bold is the most common alternative to Normal. Note that not all fonts include all weight variants; missing weights fall back to the nearest available.
FontStyle Normal / Italic / Oblique Italic uses a dedicated italic font face when available; Oblique slants the regular face algorithmically when no italic face exists. Most fonts only provide Normal and Italic.
FontStretch Normal / Condensed / Expanded / etc. Sets the horizontal stretch factor of the font. Values range from UltraCondensed to UltraExpanded. As with FontWeight, the font must include the specific stretch variant; otherwise WPF scales the nearest available face.
FontSize double The size of the text in device-independent units (1/96 inch). A value of 12 is approximately 9 points. Bind this to a ViewModel property or a resource to support dynamic font size scaling across the application.
FontFamily string The typeface to use. Accepts a font family name (e.g., "Segoe UI"), a comma-separated fallback list, or a URI pointing to a font file embedded in the application. Use font families consistently via application-level resources.
Background Brush Fills the entire TextBlock area, including padding. Useful for creating highlighted text spans. Note that setting a Background makes the TextBlock opaque in that region.
Foreground Brush Sets the text color. Any WPF brush is valid — SolidColorBrush for flat colors, LinearGradientBrush for gradient text effects. Inherits from the parent element if not set explicitly.
LineHeight / LineStackingStrategy double / BlockLineHeight / MaxHeight LineHeight overrides the font's natural line spacing for uniform multi-line layout. BlockLineHeight applies the same height to every line; MaxHeight uses each line's tallest glyph as the line height, which is more natural for mixed-size text.

XAML Example

TextBlock with rich inline formatting and text trimming with tooltip fallback:

<!-- Plain bound text with wrapping -->
<TextBlock Text="{Binding Description}"
           TextWrapping="Wrap"
           FontSize="14"
           Foreground="#333333"
           LineHeight="22" />

<!-- Trimmed text with tooltip showing full content -->
<TextBlock Text="{Binding LongTitle}"
           TextTrimming="WordEllipsis"
           ToolTip="{Binding LongTitle}"
           Width="200" />

<!-- Inline formatting -->
<TextBlock>
  <Run Text="Status: " />
  <Bold><Run Text="Active" Foreground="Green" /></Bold>
  <Run Text=" (last checked 5 min ago)" Foreground="Gray" />
</TextBlock>

Common Use Cases

  • List item labels — Display item names, descriptions, or metadata in DataTemplate cells with controlled wrapping and trimming.
  • Status bars and notifications — Show short status messages at the bottom of the window, dynamically bound to a ViewModel property.
  • Formatted paragraphs — Use inline elements to mix bold, italic, hyperlinks, and colored spans in a single text block.
  • Headings and titles — Apply larger FontSize and FontWeight="Bold" for section headers and window titles.
  • Tooltips with formatted content — Nest a TextBlock inside a ToolTip for multi-line, formatted tooltip text beyond a plain string.

Tips and Best Practices

  • Prefer TextBlock over Label for read-only text — TextBlock is lighter-weight and does not add a border or padding; use Label only when you need access-key (mnemonic) support.
  • Use Run elements for partial text binding — Bind individual Run.Text properties to update portions of a mixed-format TextBlock without rebuilding the entire Inlines collection.
  • Define fonts via resources — Centralize FontFamily, FontSize, and Foreground in application or window resources to maintain a consistent typographic scale.
  • Combine TextTrimming with a ToolTip — Always provide the full text in a ToolTip when trimming, so users can read the complete content on hover.
  • Use LineHeight for dense tables — Setting a consistent LineHeight across all cells in a data grid ensures rows align perfectly regardless of content variations.

Related Controls

  • Label — Adds access-key (underscore mnemonic) support and a Target property; heavier than TextBlock.
  • TextBox — The editable counterpart for user input.
  • ToolTip — Commonly paired with trimmed TextBlocks to show full content on hover.
